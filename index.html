03 Setup ~4 min

Settings Tour

Standards, tile positions, soffit offsets, families, custom spacings. Set up once per project.

TTell
  • Before I run Tandm, there are a few settings to set up. You only do this once per project.
  • Standards. Tandm ships with AS 2118, NFPA 13, and British Standards. You can work in metric or imperial.
  • Tile positions. You can tell Tandm which positions on a tile are valid for placement. Most of the time, I'd leave everything on. If you physically can't place in the corners of a square tile, you deactivate those positions.
  • Custom tile sizes. If the architect uses a non-standard tile size, you add it here so Tandm can match it.
  • Soffit offset for exposed areas. This sets how far off the soffit your sprinklers sit. We've defaulted to the standard maximum, 300mm for combustible, 450mm for non-combustible. You can adjust if you want to lift them to align with your range pipework.
  • Sprinkler families. We ship with Tandm families and some Victaulic families because of our partnership. I always recommend you use your own families. Load them into Revit like normal, and as long as they're unhosted, they appear in this list.
  • Custom hazard classifications. Need different spacings for extended coverage, residential, flush, semi-recessed? Create a new class based on an existing one, edit any field. You can map different families to different classes.

07 The Magic ~5 min

Splitting & Complex Rooms

Tandm places one big grid across each room. For corridors, L-shapes and rotated rooms, split into rectangles. Probably the most-used tool.

TTell
  • Tandm's engine works best in rectangles. For complex rooms, corridors, L-shapes, anything non-rectangular, you split them into rectangular zones.
  • Think of it as how you'd already break a room down in your head as a designer. Tandm just makes that explicit.
  • The splitting tool is probably the tool you'll use the most.
  • You can also use splits to force alignment across zones, re-split to centre a sprinkler row exactly where you want it.
  • For rotated or angled rooms, Tandm always orients the grid to the longest wall. So rotation doesn't matter.
  • For odd-shaped rooms, create the biggest rectangle you can, then use Free Edit to fill in the rest.

08 The Magic ~4 min

Orange Walls

Two reasons Tandm flags an orange wall. Multiple ceiling heights (Ceiling Trace), or architect turned room bounding off (Wall Trace).

TTell
  • When you see orange walls, Tandm is telling you something needs your attention.
  • Reason 1, multiple ceiling heights in one room (or a bulkhead):
    • Tandm's first pass treats the room as flat, then flags the nib wall between the heights.
    • Two ways to resolve: push sprinklers far enough to spray below the obstruction, OR use Ceiling Trace.
    • Ceiling Trace breaks the room into zones at each ceiling height and re-lays out the sprinklers per zone.
  • Reason 2, architect turned room bounding off:
    • Common in residential apartments, internal walls don't bound rooms, so Revit sees one big space.
    • I'll hit Wall Trace. Tandm rebuilds the walls and re-lays out the sprinklers.
  • In both cases, Tandm is being honest with you about what it sees, then giving you the tool to fix it in one click.
SShow
  • Multiple ceiling heights: find a room with orange walls. I'll toggle ceiling-height visibility, show the gradient (lightest highest, darkest lowest).
  • I'll show Grid Edit option: push sprinklers away until they turn blue, meaning spray below the bulkhead.
  • Then I'll show Ceiling Trace: one click, re-laid out per zone.
  • Room bounding off: find an apartment with orange walls but no ceiling step.
  • I'll hit Wall Trace. Watch the walls and layout regenerate.

Redesign Workflow

Mech change? Architect moved a wall? Select the room or group, re-run Tandm just on those. Avoids new obstructions. Margin protected.

TTell
  • Changes happen on every project. New mech layout, architect moves a wall, ceiling type changes mid-project.
  • Tandm doesn't rebuild the whole floor for a change in one room. I'll select the room (or group), hit the Fire Designer button.
  • Tandm asks if you want to delete the existing sprinklers, say yes.
  • Then I'll Place Sprinklers. Tandm re-runs avoiding the new obstruction.
  • This is also how you experiment with hazard class changes, or test a different design approach on a single room.
  • Workflow caveat. If you've grouped your sprinklers into a Revit model group (common on typical floors), re-running breaks the group. So I'd suggest using Tandm for the initial layout on those, and Revit's normal workflow for subsequent tweaks.
  • Pipework caveat: if you've connected pipework, re-running deletes the sprinklers, you'd need to reconnect. So re-runs are best for layout changes, not single-sprinkler nudges.

Next: The limitations conversation, be honest, be confident
13 Wrap ~3 min

Limitations

The honesty stack. Name them. Explain why. Roadmap. Dev team. Portfolio reframe.

TTell
  • We're upfront about where Tandm is right now. Three things to flag:
  • 1. Native Revit only.
    • IFCs and Archicad exports don't map cleanly into Revit rooms.
    • Eight to nine developers full time. IFC and 2D CAD workflows are the next major workstream after Dry Fire MVP.
  • 2. Sloping ceilings.
    • Depends on how they're modelled. Hit and miss. Not advertised.
    • Same roadmap workstream as the geometry rewrite.
  • 3. Interconnected levels.
    • Revit rooms are associated to one level. As soon as a space spills over, Tandm can't see it.
    • Car parks are the classic example, multiple levels in one big space.
    • Same roadmap.
  • Concealed space sprinklers, not native today. There are workarounds, but it's also pipework-dependent, so it's queued behind the pipework roadmap.
  • We don't do pipework yet. That's the next major build after IFC and 2D, likely 12-plus months out. Right now Tandm gives you more time for pipework, rather than doing it for you.
SShow
  • If they ask about a specific limitation in detail, I'll offer to send the project file to Byron after the call. He'll do a quick assessment of Tandm fit before they commit.
  • If they ask about IFC specifically, I'll mention the active geometry-rewrite workstream.
RReinforce
  • The portfolio reframe (this is the close on limitations):
  • Where Tandm fits in your portfolio is the projects with native Revit models and standard floor plates, that's where it'll save you days.
  • Projects with sloping ceilings, IFCs, or interconnected levels stay manual today. We're actively building toward them.
  • Tandm doesn't have to do every project to be worth it, it just has to land brilliantly on the ones where it fits.
  • The honesty stack pattern (use it for every limitation):
    • Name the limitation plainly.
    • Explain the technical why.
    • State the roadmap commitment.
    • Reference the active dev team (8-9 developers).
    • Reframe to where it does fit.
